## Architecture Context

### Project Structure Pattern

This project follows the **single-page application (SPA) pattern** with direct DOM manipulation:

```
index.html (root)
    ├─ <script type="module" src="/src/main.js">
    └─ <canvas id="app"> (generated by Three.js renderer)

src/main.js
    ├─ THREE.Scene
    ├─ THREE.PerspectiveCamera
    ├─ THREE.WebGLRenderer
    ├─ THREE.Mesh (BoxGeometry + MeshStandardMaterial)
    ├─ THREE.AmbientLight
    ├─ THREE.DirectionalLight
    └─ OrbitControls
```

### Three.js Scene Graph Hierarchy

The standard Three.js architecture requires this object hierarchy:

1. **Scene** — Root container for all 3D objects
2. **Camera** — Defines viewpoint (PerspectiveCamera with FOV, aspect, near, far)
3. **Renderer** — WebGLRenderer that draws to canvas
4. **Mesh** — Geometry + Material combination added to scene
5. **Lights** — Added to scene to illuminate materials
6. **Controls** — Attached to camera, updates via animation loop

### Rendering Loop Architecture

Three.js requires a **recursive animation loop** using `requestAnimationFrame`:

```javascript
function animate() {
  requestAnimationFrame(animate);
  
  // Update transformations (rotation, position)
  mesh.rotation.x += delta;
  mesh.rotation.y += delta;
  
  // Update controls (processes user input)
  controls.update();
  
  // Render frame
  renderer.render(scene, camera);
}
```

**Critical Timing Detail:** `requestAnimationFrame` synchronizes with the browser's repaint cycle (~60fps). Rotation increments must be small (e.g., 0.01 radians) to achieve smooth animation.

### Responsive Canvas Pattern

Full-viewport canvas requires two integration points:

**CSS (`src/style.css`):**
```css
body, html { margin: 0; padding: 0; overflow: hidden; }
canvas { display: block; width: 100vw; height: 100vh; }
```

**JavaScript (resize handler in `src/main.js`):**
```javascript
window.addEventListener('resize', () => {
  camera.aspect = window.innerWidth / window.innerHeight;
  camera.updateProjectionMatrix();
  renderer.setSize(window.innerWidth, window.innerHeight);
});
```

**Failure Point:** Forgetting `camera.updateProjectionMatrix()` causes stretched/distorted rendering after resize.

## Pattern Library

### Three.js Initialization Pattern (Established Standard)

Based on Three.js documentation and community conventions:

```javascript
// 1. Scene setup
const scene = new THREE.Scene();
scene.background = new THREE.Color(0x1a1a2e);

// 2. Camera configuration
const camera = new THREE.PerspectiveCamera(
  75,                                    // FOV
  window.innerWidth / window.innerHeight, // Aspect ratio
  0.1,                                    // Near clipping plane
  1000                                    // Far clipping plane
);
camera.position.z = 5; // Position to see origin

// 3. Renderer setup
const renderer = new THREE.WebGLRenderer({ antialias: true });
renderer.setSize(window.innerWidth, window.innerHeight);
document.body.appendChild(renderer.domElement);

// 4. Geometry and material
const geometry = new THREE.BoxGeometry(1, 1, 1);
const material = new THREE.MeshStandardMaterial({ color: 0x00ff00 });
const cube = new THREE.Mesh(geometry, material);
scene.add(cube);

// 5. Lighting
const ambientLight = new THREE.AmbientLight(0xffffff, 0.5);
scene.add(ambientLight);
const directionalLight = new THREE.DirectionalLight(0xffffff, 0.8);
directionalLight.position.set(5, 5, 5);
scene.add(directionalLight);

// 6. Controls
const controls = new OrbitControls(camera, renderer.domElement);

// 7. Animation loop
function animate() {
  requestAnimationFrame(animate);
  cube.rotation.x += 0.01;
  cube.rotation.y += 0.01;
  controls.update();
  renderer.render(scene, camera);
}
animate();
```

### Line Count Optimization Patterns

To stay under 100 lines:
- **No comments** — every line must be code
- **Chain method calls** where appropriate: `scene.add(cube).add(light)`
- **Inline object literals** for simple configurations
- **Single `window.addEventListener('resize', ...)` handler** combining all resize logic
- **Avoid abstractions** — no helper functions, no classes

### Naming Conventions

Three.js community standard names:
- `scene` — not `myScene` or `threeScene`
- `camera` — not `cam` or `perspectiveCamera`
- `renderer` — not `webglRenderer` or `render`
- `controls` — not `orbitControls` or `cameraControls`
- `animate` — not `render` or `loop` (reserved for renderer.render)

### Three.js Version Evolution Context

**OrbitControls Import Path Breaking Change:**
- **Three.js r127 and earlier:** `'three/examples/js/controls/OrbitControls'`
- **Three.js r128+:** `'three/examples/jsm/controls/OrbitControls.js'`
- **Current requirement (0.160.0):** Must use `/jsm/` path with `.js` extension

This breaking change has caused widespread project breakage in the Three.js ecosystem. Any generated code must use the modern import path.
